      Remarks

      Parameters are exposed throughs Parameters object. An SQL parameter named @PValue is created by either

      • Simply assigning the value to Parameters collection, as in Parameters.PValue = 12;. This creates an input-only parameter and works only for simple values (i.e. not strings or byte arrays).
      • By preconfiguring the parameter through QdbValueAttributes, as in Parameters.PValue += new DbPathComponentsAttributes();. Only size, type and direction are read from the attributes, and the parameter name is taken from the member name (PValue).
      Once a value is assigned to the parameter, it can be reassigned, but its type must be the same as on the first assignment. Parameter values are read by simple acces, e.g., var v = Parameters.PValue;.

      Parameters for the command can also be created explicitly with CreateParametersTParameters(EntityAccessorTParameters) method.
